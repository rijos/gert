Architecture

Design principles

  1. No central application database. No users table. Identity lives in the IdP; per-user data lives on disk. The API is a stateless translator between JWTs and folders.
  2. Isolation is a filesystem property, not a query filter. A connection only ever opens this user's files - there is no WHERE user_id = ? to forget. The same property nests one level down for projects.
  3. The user key is derived only from the token. sha256(iss + sub) from the validated JWT, never from a route, query, or body - IDOR is structurally impossible.
  4. Lazy provisioning. Folders and schemas are created on first authenticated request. A valid token is sufficient to exist.
  5. Deletion is rm -rf. Nothing references a user outside their folder.
  6. Validation is a fail-closed input boundary. Every user-supplied DTO passes a registered validator before any service acts on it; an input type with no validator is a bug, not an exemption.

Solution layout

The compiler enforces inward dependencies. Everything points inward to Gert.Service -> Gert.Model; the host and the turn engine sit outward of the service layer (host -> Gert.Agent -> Gert.Service), and the service layer references only the capability contracts, never the impl leaves - so the host (API) and the tests drive the same business logic. Each config-selected capability (chat, the database engine, the RAG engine, search, the sandbox) is split into a contracts assembly and one or more per-impl leaves:

Tech stack

  • .NET 10 / ASP.NET Core - warnings as errors, nullable strict.
  • SQLite via Microsoft.Data.Sqlite + Dapper. The database engine (Gert.Database.Sqlite) backs user.db + chat.db; the RAG engine (Gert.Rag.Sqlite) backs rag.db with sqlite-vec (vector KNN) + FTS5 (BM25) for hybrid search - both per project.
  • Microsoft.Extensions.AI (IChatClient/IEmbeddingGenerator) over the official OpenAI .NET SDK - any OpenAI-compatible server (vLLM in the reference deployment).
  • VanJS SPA - native ES modules, vendored libs, no npm, no build step; on publish a pinned, SHA-512-verified esbuild Go binary bundles it (no Node).
  • Polly resilience on all upstream HTTP; Serilog NDJSON logs (no tokens, no content).
  • Channels + BackgroundService for the detached turn and ingestion workers.
