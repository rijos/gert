Configuration

Every server-level knob, where it binds from, and what it defaults to. Standard ASP.NET Core layering applies — last one wins:

appsettings.json → appsettings.{Environment}.json → environment variables → command line
  • Environment variables — replace : with __: Gert__Vllm__BaseUrl=http://vllm:8000. Array entries by index: Gert__Models__0__Id=qwen36.
  • Command line — dotnet run --project src/Gert.Api -- --Gert:Vllm:BaseUrl=… (highest precedence).
  • Secrets never go in appsettings.json — the vLLM bearer key arrives via env vars or dotnet user-secrets.

Gert:Vllm — the model upstream

One OpenAI-compatible upstream serves both chat and embeddings.

KeyDefaultNotes
BaseUrlhttp://localhost:8000Server base URL, without /v1.
ApiKey(unset)Secret — env / user-secrets only. Sent as Authorization: Bearer.
ChatModelIddefaultSent as model on /v1/chat/completions. Must exist on the server (GET /v1/models).
EmbeddingModelIdbge-m3Sent as model on /v1/embeddings. Document upload fails without it.
EmbeddingDimensions1024Must match the embedding model. Effectively immutable once data exists — it bakes into every rag.db.
RequestTimeoutSeconds120Per-attempt timeout; Polly retries wrap it.
RetryCount2Retries on transient upstream failure.

vLLM prefix caching. Gert's requests are prefix-cache friendly: a static system prompt per project, history replayed verbatim, deterministic tool specs, no per-request unique fields. Make sure your vLLM has automatic prefix caching enabled (it's on by default in the V1 engine) and watch vllm:prefix_cache_hit_rate.

Gert:Models — the model catalog

What GET /api/models publishes to the picker, in configured order. When absent, the catalog falls back to a single entry built from Gert:Vllm:ChatModelId.

KeyRequiredNotes
IdyesSent upstream as model.
NameyesDisplay name in the picker.
DefaultnoThe server-level default. Flag exactly one.
Capabilitiesno"tools" is load-bearing — it gates tool calling. Unset means permissive; an explicit list without "tools" disables tools for that model. "vision" gates image input.
ContextnoContext window in tokens (the "128K ctx" badge). vLLM reports it as max_model_len.
FastnoDisplay-only "· fast" marker.
EndpointnoDisplay-only endpoint hint.

Users pick from this catalog; they can never add endpoints or model ids of their own.

Auth + Storage — identity and the data root

KeyNotes
Auth:AuthorityThe OIDC issuer. JWTs are validated against its JWKS, RS256 only.
Auth:AudienceExpected aud claim, e.g. gert-api.
Storage:DataRootFilesystem root holding the users/ tree. Back up this directory and you've backed up Gert.
Storage:ExpectedIssuerFail-closed iss assertion checked before any user folder is created. Normally equal to Auth:Authority.

Both Storage values are required — the host refuses to start without them.

Gert:Turn — the detached turn pipeline

A round is one upstream completion request that comes back with tool calls — executing them and re-prompting starts the next round, so every round costs a full vLLM completion.

KeyDefaultNotes
MaxTurnDuration00:05:00Hard wall-clock cap on one turn (model rounds + tools). Doubles as the orphan horizon for abandoned streaming rows.
MaxToolRounds16Hard cap on tool rounds per turn. The todo-driven flow consumes a round per set_todos update — size this to your longest expected checklist. Past the cap the runner refuses further calls and winds the turn down.
DeltaFlushInterval150msDelta coalescing window — buffered chunks emit as one event per window.
DeltaFlushMaxChars512Size backstop for the coalescing window.
