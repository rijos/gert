REST API

REST API

All endpoints require a bearer token except GET /healthz. The user is always implicit from the token - there is no userId in any path. Most data is project-scoped under /api/projects/{pid}/..., where pid is a project UUID or the literal default.

Models & settings

MethodPathNotes
GET/api/modelsThe configured model catalog: id, name, capabilities, context window.
GET/api/settingsUser preferences: theme, languages, default model/tools, memory mode.
PUT/api/settingsUpdate any subset.

Projects

MethodPathNotes
GET/api/projectsList: id, name, counts, updated_at.
POST/api/projects{ name, description?, instructions?, defaults? } -> a new isolated project folder.
GET/api/projects/{pid}Config + counts.
PATCH/api/projects/{pid}Rename / edit instructions / defaults.
DELETE/api/projects/{pid}rm -rf projects/{pid} - chats and documents. default is emptied, not removed.

Conversations

MethodPathNotes
GET.../conversationsList (id, title, model, updated_at, archived).
POST.../conversationsNew conversation; unset fields inherit project/user defaults.
GET.../conversations/{id}Full thread: messages + tool calls + citations + artifacts.
PATCH.../conversations/{id}Rename / switch model / toggle tools / archive.
DELETE.../conversations/{id}Cascade-deletes everything in the thread.

Sending a message (detached turn)

POST /api/projects/{pid}/conversations/{id}/messages

{ "content": "should I use Qdrant or sqlite-vec?",
  "model_id": "qwen36",
  "tools": { "rag": true, "search": true, "sandbox": false } }

Optional attachments carry up to 6 inline images (vision input). Responds 202 Accepted with the persisted ids and the subscribe cursor; a second POST while a turn is streaming returns 409 Conflict.

{ "conversation_id": "...", "user_message_id": "...",
  "assistant_message_id": "...", "seq": 42 }

Receiving a turn

Every event carries a per-conversation monotonic seq. Three delivery views over the same durable event log - pick by capability, switch freely:

GET .../conversations/{id}/events?after={seq}&limit={n}   # range / poll
GET .../conversations/{id}/stream?after={seq}             # SSE (live)
GET .../conversations/{id}/ws                             # WebSocket (live + cancel)
EventUI effect
message_startCreates the assistant bubble.
tool_callRenders a tool card with a spinner.
tool_resultFills the card: document hits, stdout, or the todo checklist.
deltaStreamed text append.
citationThe [1] marker + footnote.
artifactOpens a canvas tab.
message_end / cancelled / errorFinalizes the row (token count / stopped marker / inline error).

Every event is persisted before it is published, so a reload mid-turn resubscribes from the last seq and loses nothing. POST .../cancel stops an in-flight turn server-side.

Documents & memory

MethodPathNotes
GET.../documentsList: name, size, chunk_count, status, error.
POST.../documentsMultipart upload -> 202, ingestion enqueued.
GET.../documents/{id}Polled while processing - drives the status pills and progress.
DELETE.../documents/{id}Deletes chunks + vectors + FTS rows + the original file.
GET.../memoryList entries (id, title, pinned, updated_at).

Account & data lifecycle

MethodPathNotes
POST.../forget-documentsWipe one project's RAG data, keep its chats.
GET.../exportDownload one project: conversations + original files.
GET/api/account/exportDownload everything - all projects.
DELETE/api/accountrm -rf users/{key} - erases all of this user's data.

Admin (requires the Admin policy)

MethodPathNotes
GET/api/admin/usersLists user folders: username, key, size, doc count, last active.
GET/api/admin/users/{key}One user's folder summary.
DELETE/api/admin/users/{key}Removes all of that user's data. {key} must match ^[0-9a-f]{64}$ and resolve under /data/users/.

The API cannot create users - the IdP does. Admin is purely data lifecycle and usage visibility. vLLM, SearXNG, and the sandbox are never proxied to the browser; they are reached only server-side inside the chat orchestration.
