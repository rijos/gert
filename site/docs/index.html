Gert

Gert is a privacy-first, self-hosted LLM chat server built on .NET. It pairs a streaming chat UI with retrieval over your own documents, sandboxed code execution, and web search - against any OpenAI-compatible model server you run yourself, such as vLLM.

The one-line architecture: the IdP owns identity, the filesystem owns data, and the API owns nothing persistent of its own. Every user is one folder on disk; deleting a user is deleting that folder.

Features

  • No central database - there is no users table. Per-user data lives in per-user folders holding per-project SQLite files. See Storage & Isolation.
  • Hybrid RAG - vector KNN (sqlite-vec) fused with BM25 (FTS5) via Reciprocal Rank Fusion, with citations back to file and page. See Chat, RAG & Tools.
  • Five built-in tools - document search, web search (SearXNG), sandboxed Python (gVisor), a model-managed todo list, and a clock - gated per user by a JWT entitlement.
  • Artifacts canvas - named code fences become persistent canvas tabs with live HTML preview.
  • Projects & memory - isolated workspaces, each with its own chats, documents, and memory notes.
  • Detached turns - generation runs on a background worker and survives client disconnects; resume over SSE or polling from an exact event cursor.
  • Passkey login - OIDC Authorization Code + PKCE against Pocket ID; RS256-pinned JWT validation. See Security Model.
  • Hardened by structure - IDOR is structurally impossible, uploads parse in an unprivileged subprocess, the web fetcher is SSRF-guarded, and the SPA ships with a strict CSP and zero npm dependencies.
