Quick Start

From clone to chatting in four steps. Gert is one ASP.NET Core app that serves both the API and the web UI — there is no separate frontend build.

Prerequisites

  • .NET 10 SDK — the only build requirement. No npm, no Node.
  • An OpenAI-compatible model server — e.g. vLLM serving a chat model and an embedding model (default bge-m3).
  • An OIDC provider — Gert is built against Pocket ID (passkey login), but any issuer with a JWKS endpoint works.
  • Optional: SearXNG for web search, gVisor (runsc) for the Python sandbox.

1. Clone and build

git clone https://github.com/rijos/gert
cd gert
dotnet build Gert.sln -c Release

2. Configure

Edit src/Gert.Api/appsettings.json (or use environment variables — see Configuration):

{
  "Gert": {
    "Vllm": {
      "BaseUrl": "http://192.168.1.50:8000",   // NO trailing /v1
      "ChatModelId": "qwen36",
      "EmbeddingModelId": "bge-m3",
      "EmbeddingDimensions": 1024
    },
    "Search": { "BaseUrl": "http://localhost:8080" }
  },
  "Auth": {
    "Authority": "https://id.example.com",
    "Audience": "gert-api"
  },
  "Storage": {
    "DataRoot": "/data",
    "ExpectedIssuer": "https://id.example.com"
  }
}

The /v1 gotcha: Gert:Vllm:BaseUrl is the server base (http://host:8000), not the OpenAI API base. Gert appends /v1/chat/completions and /v1/embeddings itself, so a pasted /v1 suffix would double up.

If your model server needs a key, keep it out of the config file:

dotnet user-secrets --project src/Gert.Api set "Gert:Vllm:ApiKey" "sk-…"
# or
export Gert__Vllm__ApiKey=sk-…

3. Run

dotnet run --project src/Gert.Api -c Release

Open the printed URL. You'll be redirected to your IdP for passkey login; on the first authenticated request Gert lazily provisions your folder, settings, and a default project — a valid token is sufficient to exist.

4. Chat

  • Upload documents in the knowledge panel — ingestion runs in the background and shows live processing → ready status.
  • Toggle Use my docs and ask a question — answers cite file and page.
  • Ask for code or a page — named fences open as canvas tabs with live preview.

Try it without any infrastructure

The repository ships a fully mocked environment — fake IdP, fake model, fake search — so you can click through the real UI with zero setup beyond uv:

make serve-mock              # mocked everything, prints a signed-in URL
make serve-mock-vllm VLLM_URL=http://host:8000/v1   # real model, mocked auth

Console host

Gert's service layer is host-agnostic: Gert.Console runs the same chat, RAG, and ingestion services as a single-user CLI with no API or IdP at all — handy for local use and scripting:

dotnet run --project src/Gert.Console
