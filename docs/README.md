# Gert documentation

Two audiences, two folders:

- **[design/](design/README.md)** — the design docs: architecture, security, storage, the
  API, the SPA, testing. The source of truth for how the system works and why; start at its
  README, which routes you to the right doc per kind of change.
- **[installation/configuration.md](installation/configuration.md)** — the operator's
  reference: every server-level knob (`appsettings` / env), where it binds from, with working
  examples.

Related, outside this folder:

- [`site/`](../site/) — the public website, published at <https://rijos.github.io/gert/>.
